## Config Tab UI/UX Improvements — Group, Filter & Organize Tables

### Context

Currently the Config tab renders all tables as a flat list of expanders. At 20+ tables this becomes unwieldy — no grouping, no filtering, no way to quickly identify which tables need attention.

The existing app uses:
- Dark-themed custom HTML cards with CSS classes (`.metric-card`, `.table-card`, `.pill`)
- `st.columns`, `st.expander`, `st.tabs` for layout
- Source schema is stored as `tbl["source_db"]` per table
- Status/review info available: `active`, `last_run_status`, `_review`, `load_type`

### Implementation Steps

#### 1. Add summary hero cards above the table list

Before the table expanders, show 4 small metric cards:
- **Active** (count of `active=True`)
- **Inactive** (count of `active=False`)
- **Needs Review** (count of tables with `_review` key)
- **Total** tables

This gives immediate visibility into what needs attention.

#### 2. Add filter bar (search + filter controls)

A row with:
- **Text search** — filters table list by name (source_table or target_table)
- **Schema filter** — multiselect dropdown of distinct `source_db` values (groups by schema)
- **Status filter** — selectbox: All / Active / Inactive / Needs Review

#### 3. Group tables by schema

Tables are grouped under schema sub-headers (`st.markdown` section dividers). Each schema group shows:
- A collapsible section header: `📂 employees (4 tables)` or `📂 sales (12 tables)`
- Tables within that group rendered as expanders (existing behavior)

#### 4. Add bulk actions row per schema group

Above each schema group, a row with:
- **Activate All** button — sets all tables in the group to `active=True`
- **Deactivate All** button — sets all tables in the group to `active=False`

#### 5. Compact table label with status indicators

Enhance the expander label to show more info at a glance without expanding:
```
🟢 orders → ORDERS [incremental] ⚠️ review  |  ✅ success  |  🕐 2026-06-10
```

### Verification

  - Search filters the list in real-time
  - Schema grouping shows correct headings

  - Existing per-table editing still functions
  - Works with 0, 1, 6, and 20+ tables
